SMS Consent Verification

Public proof of opt-in flow for 10DLC campaign compliance

Purpose of This Page

This page provides publicly verifiable documentation of how Prime Anchorpoint collects SMS consent from end users. It is intended for carrier and campaign registry reviewers to verify our Call to Action (CTA) and opt-in compliance.

Brand: Prime Anchorpoint  |  Campaign: Workforce Notifications (Mixed - OTP, Shift Reminders, HR Alerts)

1. Worker Registration — Primary Opt-In (Explicit Checkbox)

Live Registration Page

Workers register at the following publicly accessible URL:

https://www.primeanchorpoint.com/register

During registration, the worker must provide a phone number and explicitly check an unchecked SMS consent checkbox before the form can be submitted. The checkbox is not pre-checked. The exact consent language displayed is:

This consent language is displayed in all three supported languages (English, Chinese, Spanish) based on the user's language selection.

Opt-In Flow (Step by Step)

1
User visits /register and selects "Worker" registration type
2
User fills in name, email, phone number, and other required fields
3
User checks the SMS consent checkbox (unchecked by default). The full disclosure text including STOP/HELP instructions, frequency, rates, and links to SMS Terms & Privacy Policy is displayed directly next to the checkbox.
4
User submits the registration form. The consent status (sms_consent: true/false) and a timestamp are recorded in our database.
5
System sends an OTP verification code via SMS only if consent was granted. No messages are sent without opt-in.

2. Website Contact / Job Inquiry Form — Secondary Opt-In

Contact Form on Homepage

The website homepage (https://www.primeanchorpoint.com) includes a contact form where users can submit inquiries. SMS disclosure language is displayed next to the submit button:

This form is used for employer inquiries, job seekers, and general questions. SMS is only sent in direct response to the user's inquiry.

3. Message Types and Frequency

We send only transactional / operational messages

  • OTP / Verification Codes: Account registration, login, identity verification
  • Job Assignment Alerts: New assignment details including location, time, and supervisor
  • Shift Reminders: Upcoming shift start reminders and schedule changes
  • Onboarding / HR Updates: Document submission reminders, background check status, compliance deadlines
  • Interview Notifications: Interview scheduling and reminders

We do not send marketing or promotional messages. All SMS is transactional and directly related to the user's employment or application.

Message frequency varies based on registration activity, active assignments, and onboarding status. Typically 1–5 messages per event, up to 10 per week during active onboarding periods.

4. Opt-Out and Help

  • Users can reply STOP at any time to opt out of all messages
  • Users can reply HELP to receive program information and support contact
  • Opt-out and help keywords are processed automatically
  • All messages include opt-out instructions
