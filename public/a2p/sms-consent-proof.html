Recurring transactional SMS (job assignment alerts, shift reminders, onboarding updates, interview notifications, payroll alerts, and responses to direct inquiries) are sent only to users who checked the consent checkbox. One-time verification codes (OTP) are separate — they are user-initiated security messages sent only when the user triggers an account action (registration, login, identity verification) and do not require the consent checkbox.

2. Verification Codes (OTP) vs. Recurring SMS

Prime Anchor Point sends two categories of text messages. It is important to distinguish between them:

  • Recurring transactional SMS — Job assignment alerts, shift reminders, onboarding updates, interview notifications, payroll alerts, and responses to user inquiries. These are sent only after the user explicitly opts in by checking the SMS consent checkbox.
  • One-time verification codes (OTP) — Account registration verification, login verification, and identity verification codes. These are sent only when the user initiates an account action (such as registering, logging in, or verifying identity). OTPs do not require the SMS consent checkbox and are standard security messages triggered by user action.

Opting out of recurring SMS via STOP does not prevent one-time verification codes that the user initiates.

3. Message Types and Frequency

Recurring Transactional SMS (require SMS consent checkbox)

  • Job Assignment Alerts: New assignment details including location, time, and supervisor
  • Shift Reminders: Upcoming shift start reminders and schedule changes
  • Onboarding Updates: Document submission reminders, background check status, and onboarding step completions
  • Interview Notifications: Interview scheduling confirmations and reminders
  • Payroll Alerts: Pay period reminders and direct deposit notifications
  • Direct Responses to User Inquiries: Replies to messages or inquiries submitted by the user

One-Time Verification Codes (OTP — do NOT require the consent checkbox)

  • Account Registration Verification: One-time code sent when a user registers
  • Login Verification: One-time code sent when a user signs in
  • Identity Verification: One-time code sent during identity confirmation steps

OTPs are standard security messages triggered solely by user-initiated account actions. They are independent of the recurring SMS consent checkbox.

We do not send marketing or promotional messages. All SMS is transactional and directly related to the user's employment, application, or direct inquiry.

Message frequency varies based on user activity, assignments, onboarding status, and direct inquiries.
